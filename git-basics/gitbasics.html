📚 Git Basics (The Odin Project)

🎯 Lesson Objectives

  • Create a GitHub repository.
  • Clone a repository.
  • Track changes with Git.
  • Commit project snapshots.
  • Push code to GitHub.

⚙️ Basic Git Workflow

git clone <repo-url>
git status
git add .
git commit -m "Message"
git push
git log

💡 Important Concepts

  • Repository: Your project.
  • Clone: Download repository.
  • Stage: Prepare changes before commit.
  • Commit: Save a project snapshot.
  • Push: Upload commits to GitHub.
  • Origin: Remote GitHub repository.
  • Main: Default branch.

⭐ Best Practices

  • Commit small changes.
  • Write meaningful commit messages.
  • Check git status frequently.
  • Edit locally, then push to GitHub.
  • Push your work regularly.

📝 Summary

Git helps track code changes, while GitHub stores your code online. The standard workflow is:

 ↓
